    Adding Amplify Auth¶

    Estimated time to read: 3 minutes

    Amplify Auth uses Amazon Cognito under-the-hood. Cognito is an Amazon service that is built to handle user registration, authentication, andother such operations.

    To get started, we shall define our authentication resource. Create an auth resource file amplify/auth/resource.ts

    export const auth = defineAuth({
    loginWith: {
        email
})

    By default, the auth resource is configured to use email as its login method. You can use a different configuration for your projects but for this workshop, we will be using the simplest configuration.

    After defining auth resource, pass it to your amplify/backend.ts 

    import { defineBackend } from '@aws-amplify/backend';
import { auth } from './auth/resource';

defineBackend({
  auth,
});

    Deploying Auth Resource¶

    After defining your auth resource, create your resource in your personal cloud sandbox. This deploys your backend resources in a safe, isolated environment.

    Run the following command:

    npx ampx sandbox

    Following a successful deployment, the command generates an outputs file amplify_outputs.json. This file serves as a connection between your frontend app and your backend resources.

    What happens when you run this command?

    • Amplify reads your backend definitions (like auth/resource.ts).

    • Amplify provisions the resources you defined. For Auth, this includes a Cognito User Pool, Identity Pool, and related authentication infrastructure in your sandbox environment.

    • Once deployment is successful, it generates a file called amplify_outputs.json. This contains all the configuration details and resource identifiers your frontend needs to communicate with the backend.

    Connecting your Frontend to your Auth Resource¶

    Navigate to your main app /src/App.tsx and add the following lines:

    import { Amplify } from 'aws-amplify';
import outputs from '../amplify_outputs.json';

Amplify.configure(outputs);     // Connects your backend to your frontend

    Amplify.configure(outputs) reads the outputs file and automatically sets up connections to all your deployed backend resources (in this case, authentication).

    Congratulations. Now your React App has been connected to your Amplify backend!

    Implementing Authentication¶

    Now let's add a frontend interface to implement our Authentication.

    Creating a sign-in form and implement the right flow can be time-consuming and challenging. With Amplify's Authenticator UI component, we can think less of the authentication flow. The components works along with the configuration in your amplify/auth/resource.ts to automatically connect to your backend resources.

    First step would be to install the @aws-amplify/ui-react library:

    npm add @aws-amplify/ui-react

    After successfall installation, navigate to your src/App.tsx and import the Authenticator Component and Amplify's styles.css.

    import { Authenticator } from '@aws-amplify/ui-react';
import '@aws-amplify/ui-react/styles.css';

    Then wrap your App with the Authenticator tags in your return function.

    <Authenticator>
    <main>
        <User />
        <Posts />
    </main>
</Authenticator>

    After wrapping our App with Amplify's Authenticator, a built-in Sign-up page with a configured authentication flow will be shown when we are not authenticated thanks to Amplify Authenticator UI:

    Screenshot here

    If we are able to sign-in, we should be able to sign-out as well. Navigate to components/User.tsx. This is our top component after successfuly signing-in. Using the @aws-amplify/ui-react, import the useAuthenticator hook

    import { useAuthenticator } from "@aws-amplify/ui-react";

    Use that hook to give us access to our signed user information and the signOut function. Replace lines accordingly: 

      const { user, signOut } = useAuthenticator();
  const username = user?.signInDetails?.loginId;

  return (
    <div className="user-container">
      <h1>Hi, {username}</h1>
      <button onClick={signOut}>Sign Out</button>
    </div>
  )
}

    After saving the changes, we are able to display the logged-in user's email and a working Sign Out button.
