    Set up Amplify Data¶

    Estimated time to read: 2 minutes

    With Amplify Data, you can quickly create a secure, real-time API with a database. Just define your data model in TypeScript, and Amplify sets up the API using AWS AppSync and DynamoDB. You can secure it with rules and add custom features with AWS Lambda.

    Building your data backend¶

    To get started, we shall define our data resource. Create an auth resource file amplify/data/resource.ts and fill it with the following:

    import { a, defineData, type ClientSchema } from '@aws-amplify/backend';

const schema = a.schema({
  Post: a.model({
    title: a.string(),
    email: a.string(),
    userId: a.string(),
    date: a.string(),
    imageKey: a.string()
  })
    .authorization(allow => [
      allow.authenticated().to(['read']),
    ])
});

// used for code completion / highlighting when making requests from frontend
export type Schema = ClientSchema<typeof schema>;

// defines the data resource to be deployed
export const data = defineData({
  schema,
  authorizationModes: {
    defaultAuthorizationMode: 'userPool',
  }
});

    Don't forget to pass it to your amplify/backend.ts 

    import { defineBackend } from '@aws-amplify/backend';
import { auth } from './auth/resource';
import { data } from './data/resource';

defineBackend({
  auth,
  data,
});

    Code Breakdown¶

    schema

    a is Amplify's backend builder that allow us to define schemas, models, fields, etc. With that, we defined a schema where it has a model named Post with the fields: title, email, date that will be used to populate our frontend content later.

    authorization rules 

    .authorization(allow => [
    allow.authenticated().to(['read']),
])
    This tells Amplify that only signed-in-users can read posts.

    ClientSchema

    This generates TypeScript types for the frontend. This provides type-checking and autocomplete for Amplify objects. Not required but helps with development.

    defineData

    This creates the data backend configuration: - AppSync GraphQL API - DynamoDB tables - Realtime subscriptions - API authentication using Cognito User Pools

    defaultAuthorizationMode tells Amplify which method to use when authorizing every API request. Setting it to 'userPool' means the API will require a valid Cognito User Pool token, so only authenticated users can interact with your data backend unless you define additional authorization rules.

    With your data backend now fully defined and connected to Amplify, you’re ready to start implementing create, read, update, and delete (CRUD) operations in your frontend.
